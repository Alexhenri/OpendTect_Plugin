OpendTect Pro & dGB Plugins Documentation - 6.4

Definitions:

  • OpendTect is a complete Open Source seismic interpretation system, created by dGB Earth Sciences B.V., that is released free-of-charge, also for commercial usage, under the open source GPL license. It runs without license managing software.
  • OpendTect Pro is OpendTect extended with a closed source layer that offers extra functionality for professional users. OpendTect Pro is released under the OpendTect Pro license to commercial users and under the Academic license to academic users.
  • OpendTect Pro users can further extend their system with a range of closed source plugins for special work. These plugins are developed by dGB and by other software vendors.
  • OpendTect Pro and the commercial plugins are protected by FlexNet license managing software. To obtain an evaluation license key please contact dGB
  • OpendTect Pro and the commercial licenses can be evaluated without the need for license keys when the software is run on two training data sets. F3 Demo and Penobscot are free surveys that are released by dGB via the Open Seismic Repository (OSR)
  • This document only describes the functionality in OpendTect Pro and in the plugins that are developed by dGB. For a description of functionality in OpendTect, please see the OpendTect User Documentation
  • For plugins by ARK CLS, ARK CLS and Earthworks and SITFAL please see the respective user documentations.

OpendTect Pro and Plugins by dGB  (Described in this Document.)

  • OpendTect Pro: is the commercial layer on top of OpendTect with PetrelDirect link to Petrel*, basemap + mapping, PDF3D for sharing 3D images, Thalweg tracker for seismic facies tracking and Ray-tracer for AVA Attributes and Angle Stacks.
  • HorizonCube: Game-changing plugin that generates a dense set of horizons (e.g. a seismic event every 4ms). HorizonCube impacts all aspects of seismic interpretation work and allows the interpreter to extract more geology from the data. HorizonCube is used for: detailed geologic model building, improving seismic inversion, sequence stratigraphic interpretation (SSIS) and correlating wells (Well Correlation Panel plugin).
  • Well Correlation Panel (WCP): an interactive viewer to pick well markers and correlate these from well-to-well using seismic data to guide the correlations. Provided the user has access to the HorizonCube, the HorizonCube slider can be used for detailed seismic-steered correlations.
  • Dip-Steering: allows the user to create and use "(dip-)steering cubes". A SteeringCube contains the local dip and azimuth of the seismic events at every sample position. The cube is used for:
    1. Structurally oriented filtering (e.g. dip-steered median filter)
    2. Improving multi-trace attributes by extracting attribute input along reflectors (e.g. dip-steered similarity)
    3. To calculate some unique attributes (e.g. Dip and Azimuth, 3D-Curvature, and Variance of the dip).
    4. To auto-track unconformities.
    5. To create HorizonCubes.
  • Sequence Stratigraphic Interpretation System (SSIS) is an add-on to the HorizonCube. SSIS supports Wheeler transformations and sequence stratigraphic (systems tracts) interpretations based on chrono-stratigraphic horizons from the HorizonCube. The Wheeler transformation is one of the key features of SSIS Plugin. A Wheeler transformation is a flattening of seismic data according to the calculated chrono-stratigraphy. In the Wheeler domain we see when (in relative geologic time) and where (spatially) events were deposited, how the depositional center shifted over time (basin-wards or landwards) and how events are related in time. Gaps in the Wheeler domain are caused either by erosion or non-deposition
  • SynthRock A powerful toolkit for creating and using forward models in qualitative and quantitative seismic interpretation studies by integrating Rock Physics, Geology and Seismic data SynthRock makes full use of the power of OpendTect to support a range of cutting edge modeling and inversion work flows. The following functions are supported:
    1. Rock-physics library (Castagna, Krief, Garder, Biot-Gassmann, etc.) and possibility to define any rock physics formula using math and logic.
    2. Prestack synthetics; PP, PS, optionally with multiples; Near, mid, far, full, angle, AVO and AVA attributes etc.
    3. Profile module: create cross-sections from existing wells with manual updates of model parameters.
    4. Stochastic module: stochastically varying pseudo-wells.
    5. Inversion possibilities: HIT Cube (cross-matching procedure to create probability volumes), Probability Density Functions (derived from cross-plots), and Neural Networks(non-linear approach to predict rock property volumes)
  • Fluid Contact Finder (FCF) is used to detect subtle hydrocarbon-related seismic anomalies and to pin-point gas-water, gas-oil, and oil-water contacts. FCF uses the power of stacking to enhance such anomalies. If we stack all traces along the same contour line, we can expect the hydrocarbon effect to stack up while stratigraphic variations and random noise will be canceled out. A local FCF filter is available for smoothing 4D anomalies.
  • Neural Networks Both Supervised and Unsupervised NN are supported. Unsupervised networks are used for 2D (waveform segmentation) and 3D clustering of attributes. Supervised networks are used for creating seismic object "probability" volumes (e.g. Chimney Cubes) and to predict rock properties from seismic data and well logs.
  • Velocity Model Builder (VMB): Uses the travel time of the acoustic waves to image the subsurface.
  • Faults & Fractures (FF): is a plugin to compute various fault attributes (such as Thinned Fault Likelihood, Fracture Density, ...), to extract fault planes and to un-fault the data. In combination with Dip-Steering FF features more attributes and filters (e.g. volume curvature, dip-steered similarity, dip-steered median filter, ...).

Plugins by ARK CLS (for the user doc open Help in the respective plugin.)

  • Seismic Spectral Blueing (SSB): A technique that shapes the seismic spectrum to optimize the vertical resolution without boosting noise to an unacceptable level. The spectrum is reshaped to 'match' observed behavior of reflectivity data obtained from wells. In a global sense, the well-reflectivity spectrum shows that the higher the frequency the higher the amplitude. We refer to this as the spectrum being blued.
  • Seismic Coloured Inversion (SCI: enables rapid band-limited inversion of seismic data. A single convolution inversion operator is derived which optimally inverts the data. The spectrum of the inverted data honours the available well data spectra in a global sense. Generally, traditional inversion methods (e.g. sparse-spike) are time consuming, expensive, require specialists and are not performed routinely by the Interpretation Geophysicist, whereas SCI is rapid, easy to use, inexpensive, robust and does not require expert users. SCI and unconstrained sparse-spike appear to give broadly equivalent results.
  • Seismic Net Pay (SNP) is an improved method to estimate net pay from seismic colour inverted data. It generates net pay and net-to-gross maps with uncertainties.
  • Seismic Feature Enhancement (SFE): is a flat-spot utility that enhances the signal of consistent flat events and reduces the "noise" of the channel reflections.
  • Frequency Shaping (formerly known as Seismic Broadband) is an innovative technique that integrates well data into your seismic analysis. By using acoustic impedance logs and reflectivity two operators are simultaneously derived, once applied, they effectively broaden the amplitude spectrum of the seismic data at both ends, low and high.
  • Spotlightcan rapidly combine up to four different coloured seismic datasets into a single cube that resembles the behaviour of a given elastic attribute. The relationship between the input data and the desired output is established from prior well log analysis.
  • Workstation Access (WA): Based on the Ideal toolkit from ARK CLS . It supports direct import and export of seismic volumes, horizons and well data to and from Landmark's SeisWorks** and Schlumberger's GeoFrame-IESX*** workstations. The Workstation Access plugin supports connection with GeoFrame v4.2 and v4.3 on Linux and connection with v4.0.4, v4.0.4.1, v4.0.4.2, v4.2 and v4.3 on Solaris.....
  • Xfield (XF): Integrated potential field modelling software built from the seismic interpreter’s perspective.

Plugins by ARK CLS and EarthWorks (for the user doc open Help in the respective plugin.)

In conjunction with

  • Deterministic Inversion includes a 3D model builder for constructing a priori impedance models using well log and seismic horizon data; a 2D error grid generation module for providing spatial inversion constraints and a model-based deterministic inversion module.
  • Multi-Point Stochastic Inversion (MPSI): includes the MPSI ultraFast stochastic inversion module for generating multiple geostatistical realizations and the utilities for processing the multiple realizations to represent the inversion uncertainty for lithology, porosity, saturation or other attributes as probability cubes. This plugin also requires the purchase of the deterministic inversion plugin.

Plugins by Estimages (for the user doc open Help in the respective plugin.)

  • MGS-Destriping (MGSD): 2D automated destriping solution based on factorial kriging technique. Runs on slices of volumes and horizons.

Plugins by The Visual Wavefield Project (for the user doc open Help in the respective plugin.)

  • Seismic Terrain Explorer (STE): The first program in the history of exploration whose sole purpose is the visual communication of seismic information.

Plugins by Geo5

  • FracTex plugin computes grey level co-occurrence matrix (GLCM) based attributes in different directions and compares these results.

Plugins by LTrace

  • Bayesian Linear Inversion provides a deterministic elastic seismic inversion using the linearized Bayesian methodology for estimating velocities or impedances and density.
